# PHASE 13: DASHBOARD STABILITY & JOB MANAGEMENT FIXES

## Dashboard Root Cause Investigation

Perform a complete audit of the scraper dashboard.

Current issues observed:

* Scrapers continue running indefinitely.
* Jobs do not automatically end.
* Status becomes stuck in Running state.
* Completion status is not updated correctly.
* Failed jobs remain active.
* Dashboard metrics become inaccurate.
* Progress tracking is unreliable.

Determine the exact root cause.

Do not apply temporary patches.

Identify and fix the actual source of the problem.

Investigate:

* Job lifecycle management
* Queue processing
* Worker execution
* Background tasks
* Database status updates
* Redis/cache issues
* Timeout handling
* Failed task cleanup
* Process termination logic
* Dashboard state synchronization

Document findings and implement permanent fixes.

---

## Automatic Job Completion

Every scraper run must have a proper lifecycle.

Required states:

Queued

Running

Quality Gate Validation

Completed

Failed

Cancelled

A scraper must automatically move to Completed when:

* Scraping is finished
* Quality Gate passes
* Shopify upload completes
* Final validation completes

A scraper must automatically move to Failed when:

* Validation fails
* Scraping crashes
* Import fails
* Configuration loading fails
* Critical errors occur

No scraper should remain permanently in Running state.

---

## Stuck Job Detection

Implement automatic stuck-job detection.

Examples:

* No progress for X minutes
* No products processed
* No worker activity
* No database updates

Automatically:

* Mark job as Stuck
* Log reason
* Alert dashboard
* Allow restart
